Iniciemos con DWR

DWR es una herramienta que te permite importar estructuras y estilos personalizados a tus proyectos, dando la posibilidad de modificarlos cada vez que gustes.

Importación

Comencemos añadiendo WDR al proyecto

1. Crea un nuevo index.html. Incluye la etiqueta <meta name="viewport"> para tener un comportamiento responsivo en dispositivos móviles.

                
                    <!DOCTYPE html>

                    <html lang="es">

                    <head>

                        <meta charset="utf-8">

                        <meta name="viewport" content="width=device-width, initial-scale=1">

                        <title>DWR demo</title>

                    </head>

                    <body>

                        <h1>Hello, world!</h1>

                    </body>

                    </html>

2. Incluye DWR. Pon la etiqueta <link> en el <head> para tener importar todos los estilos CSS.

    
            <!DOCTYPE html>

            <html lang="es">

            <head>

                <meta charset="utf-8">

                <meta name="viewport" content="width=device-width, initial-scale=1">

                <title>DWR demo</title>

            </head>

            <body>

            <link rel="stylesheet" href="https://aciel1234.github.io/design@latest/css/web-resources.css">
                <h1>Hello, world!</h1>

            </body>

            </html>

3. Abre la página en el navegador para a ver los resultados. Ahora podrás comenzar a usar y modificar los estilos en base a tus preferencias.

Links

Podrás usar este link para copiar y pegar todas las estructuras de html disponibles. De esa forma se aplicarán los estilos correspondientes a cada una.

También se añadió el atributo crossorigin="anonymous" para prevenir errores CORS.

Variables importantes

DWR utiliza variables globales importantes con el fin de normalizar el comportamiento generalizado de la web.

HTML5 doctype

DWR requiere el uso de HTML5 doctype. Sin este, podrán ocurrir errores de estilo inesperados.

            
                <!DOCTYPE html>

                <html lang="es">

                ... 

                </html>

Viewport meta

DWR tambíen está visto para el desarrollo web en dispositivos móviles, por lo que se hace necesario esta etiqueta en el head para que la página se más escalable en combinación con los componentes necesarios usando CSS media queries.

            
                <meta name="viewport" content="width=device-width, initial-scale=1">

Puedes ver un ejemplo de esta acción en Iniciemos con DWR.

Box-sizing

Para que el comportamiento de la página web sea más generalizado, en CSS se emplea box-sizing con un valor de border-box. Esto asegura que el padding no afecte el width de ningún elemento.

Si en alguna ocasión necesitas implementar otro valor, usa algo como:

            
.selector-para-algun-componente {
  box-sizing: content-box;
}

Con el ejemplo de arriba, todos los componenes hijos de la etiqueta que lleve esa clase -incluyendo los generados via ::before y ::after- tomarán el valor del box-sizing especificado.

Lee más acerca del box model en MDN.

Contenido

La librería de DWR está compuesta por:

            
                design/ 

                  |-- css/ 

                  |   |__ web-resources.css 

                  |-- img/ 

                  |  
                  | 

                  |__ index.html 

                  |__ style.css 

                  |__ about.html 

                  |__ about-style.css 

                  |__ REDAME.md 

                  |__ script.html

El archivo index.html es la página de presentación de DWR junto con about.html, mientras que dentro de la carpeta css tenemos el archivo web-resources.css, la librería de DWR, que contiene todos los estilos CSS disponibles. Entre otros archivos.
